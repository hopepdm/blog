离开其实有点突然，但是也在预期之内。
今年是ai大爆发的一年，刚过完年，claude code等agent工具强势出现，给我的工作带来了很大的便利。也给整个行业带来了恐慌。老板从中看到了巨大的降本增效机会，大力推进超级员工模式。

而我也在顺势而为积极思考，拥抱ai。但是在公司始终处理的是一些边边角角不痛不痒的业务。乱七八糟的技术栈和混乱的业务流程。

元旦后，被安排去支撑智慧医疗，又开始接触uniapp，qiankun等技术。其实这些技术都是应用层业务测的一些解决方案。但是每次都是被要求一会去vue的项目，一会去react的项目，一会去去electron或者react-native的项目，想想真的蛮崩溃。

也幸好随着工作经验的积累也不再抗拒新技术的从0-1过程。而且越来越能更好把控项目进度，这算是频繁切换技术栈的一个好处吧。

记得是hrbp在飞书上找我的时候，我立刻就领会到我可能面临的遭遇。结果如约而至，被协商一致解除劳务合同。n + 1的赔偿，年假转为薪资。hrbp还帮忙办理失业金的申请，让我感觉还行。

离开前一天和公司其他同时聊天说明了自身遭遇，聊了许多。事后想想，n 应该可以争取去年工资+奖金 / 12 作为赔偿。这样能够争取更多。我是一个比较随意的人，所以没有太在意这个。现在想想，能争取的不争取，纯属sb了是，还是经验不足。因此有了这篇文章来记录下。工作要留痕，内容要按照STAR（场景-任务-行动-结果）原则记录并汇报，将工作内容数据化，可量化。积极向上反馈，工作等成果你不说，上面就不会看见，不要傻傻的只知道埋头干活呐！！！

还有在公司5年了，交还电脑后，所有资料文件项目基本都无了。我哭死，现在一想到那个破电脑，我就觉得可笑，公司最该降本增效的就是换电脑。打工人要在职场中好好保护自己的产出成果！！！

现在已经04-20了，算算日子，快一个月了。给我的感受是这次的就业形势非常悲观，面试机会几乎为零。好点的公司还得认真准备面试，才能找到自己的位置。而且我现在这个年纪，已经没有那么年轻了。以前常常调侃35岁魔咒，等真轮到自己，只能接受现实笑嘻嘻。这里非常感谢我的宝宝，始终如一的陪伴我支持我，我很爱她^_^。

周一晚上会有一场面试，积极准备中。先写到这里吧～


ps: 好久没更新博客了，今天更新遇到点bug这里记录下

这通常说明你现在很可能“开了两套发布机制”：

一套是你仓库里的自定义 Actions workflow，文件是 .github/workflows/jekyll-docker.yml
另一套是仓库 Settings -> Pages 里仍然配置成了 Deploy from a branch，于是 GitHub Pages 自己又跑了一次内建部署
如何屏蔽：

打开 GitHub 仓库 Settings -> Pages
找到 Build and deployment
把 Source 切到 GitHub Actions
不要再用 Deploy from a branch
这样保留你自己的 workflow 后，pages-build-deployment 一般就不会再作为那种“分支发布”的内建流程重复触发了。
